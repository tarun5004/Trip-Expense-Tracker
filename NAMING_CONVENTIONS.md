## 1. File Naming

### 1.1 General Rules

| Context | Convention | Example |
|---|---|---|
| Server module files | `camelCase` with dot-separated role suffix | `expense.controller.ts`, `expense.service.ts`, `expense.repository.ts` |
| Server shared utilities | `camelCase` | `debtSimplifier.ts`, `pagination.ts`, `tokenGenerator.ts` |
| Server error classes | `PascalCase` | `AppError.ts`, `ValidationError.ts`, `NotFoundError.ts` |
| React components | `PascalCase` | `ExpenseCard.tsx`, `SplitCalculator.tsx`, `ProtectedRoute.tsx` |
| React hooks | `camelCase` with `use` prefix | `useGroups.ts`, `useCreateExpense.ts`, `useSocket.ts` |
| React stores (Zustand) | `camelCase` with `Store` suffix | `authStore.ts`, `uiStore.ts`, `socketStore.ts` |
| API client files | `camelCase` with `.api` suffix | `expenses.api.ts`, `settlements.api.ts`, `auth.api.ts` |
| Type definition files | `camelCase` with `.types` suffix | `expense.types.ts`, `api.types.ts` |
| Zod schema files | `camelCase` with `.schema` suffix | `expense.schema.ts`, `group.schema.ts` |
| Test files | Mirror source file + `.test` suffix | `expense.service.test.ts`, `debtSimplifier.test.ts` |
| Config files | `camelCase` | `database.ts`, `redis.ts`, `socket.ts` |
| Migration files | Auto-generated by Prisma | `20260329130000_create_expenses_table/` |

### 1.2 Directory Naming

All directories use `camelCase`:

```
✅  modules/expenses/
✅  shared/errors/
✅  hooks/queries/
✅  components/balances/

❌  Modules/Expenses/
❌  shared/Errors/
❌  hooks/Queries/
```

### 1.3 Index Files

- Each module's route file doubles as the public API — no `index.ts` barrel exports in modules (avoids circular dependency issues).
- `index.ts` is only used in directories that need a single entry point: `config/index.ts`, `sockets/index.ts`.

## 2. Variable & Function Naming

### 2.1 TypeScript / JavaScript

| Category | Convention | Examples |
|---|---|---|
| Variables | `camelCase` | `totalAmount`, `groupMembers`, `isAuthenticated` |
| Constants (module-level) | `UPPER_SNAKE_CASE` | `MAX_GROUP_MEMBERS`, `JWT_ACCESS_EXPIRY`, `SPLIT_TYPE_EQUAL` |
| Functions | `camelCase`, verb-first | `calculateSplits()`, `validateExpenseTotal()`, `simplifyDebts()` |
| Async functions | `camelCase`, verb-first | `fetchGroupBalances()`, `createExpense()` |
| Boolean variables | `camelCase` with `is`, `has`, `can`, `should` prefix | `isAdmin`, `hasReceipt`, `canEditExpense`, `shouldNotify` |
| Private class members | `camelCase` with `#` prefix (ES private) | `#refreshToken`, `#connectionPool` |
| Event handler props | `camelCase` with `on` prefix | `onSubmit`, `onClose`, `onExpenseCreated` |
| Callback functions | `camelCase` with `handle` prefix | `handleFormSubmit`, `handleSettleUp` |

### 2.2 Enums

TypeScript enums use `PascalCase` for the enum name and `UPPER_SNAKE_CASE` for values:

```typescript
enum SplitType {
  EQUAL = 'equal',
  EXACT = 'exact',
  PERCENTAGE = 'percentage',
  SHARES = 'shares',
}

enum SettlementStatus {
  PENDING = 'pending',
  CONFIRMED = 'confirmed',
  DISPUTED = 'disputed',
}

enum GroupRole {
  ADMIN = 'admin',
  MEMBER = 'member',
}
```

### 2.3 TypeScript Interfaces & Types

| Category | Convention | Examples |
|---|---|---|
| Interfaces | `PascalCase`, noun, no `I` prefix | `Expense`, `GroupMember`, `BalanceSummary` |
| Type aliases | `PascalCase` | `SplitRule`, `PaginationParams`, `ApiResponse<T>` |
| Generic type params | Single uppercase letter or descriptive `PascalCase` | `T`, `TData`, `TError` |
| Request DTOs | `PascalCase` with `Request` suffix | `CreateExpenseRequest`, `SettleUpRequest` |
| Response DTOs | `PascalCase` with `Response` suffix | `ExpenseResponse`, `BalanceResponse` |

## 3. Database Naming

### 3.1 Tables

| Rule | Convention | Examples |
|---|---|---|
| Table names | `snake_case`, **plural** | `users`, `groups`, `expenses`, `settlements` |
| Join tables | Alphabetical order of the two entities | `group_members` (not `member_groups`) |
| Audit/log tables | Suffixed with `_logs` | `activity_logs` |

### 3.2 Columns

| Rule | Convention | Examples |
|---|---|---|
| Column names | `snake_case` | `total_amount`, `created_at`, `display_name` |
| Primary key | `id` (UUID) | Every table has `id` |
| Foreign keys | `{referenced_table_singular}_id` | `user_id`, `group_id`, `expense_id` |
| Timestamps | `created_at`, `updated_at`, `deleted_at` | Auto-managed by Prisma |
| Boolean columns | `is_` prefix | `is_archived`, `is_recurring` |
| Enum columns | Descriptive noun, stored as `varchar` | `status`, `role`, `split_type`, `payment_method` |
| JSONB columns | Descriptive noun | `preferences`, `before_state`, `after_state` |
| Monetary amounts | `amount` or descriptive noun + explicit precision | `total_amount DECIMAL(15,2)`, `share_amount DECIMAL(15,2)` |

### 3.3 Indexes

| Convention | Pattern | Example |
|---|---|---|
| Single-column index | `idx_{table}_{column}` | `idx_expenses_group_id` |
| Multi-column index | `idx_{table}_{col1}_{col2}` | `idx_expense_splits_expense_id_user_id` |
| Unique constraint | `uq_{table}_{column}` | `uq_users_email` |
| Foreign key constraint | `fk_{table}_{referenced_table}` | `fk_expenses_groups` |

### 3.4 Migrations

- Auto-generated by Prisma Migrate.
- Naming: `{timestamp}_{description}` in `snake_case`.
- Example: `20260329130000_create_expenses_table`
- Never modify or delete existing migration files in production — always create new migrations.

## 4. API Endpoint Naming

### 4.1 URL Segments

| Rule | Convention | Examples |
|---|---|---|
| Resource names | `kebab-case`, **plural** nouns | `/groups`, `/expenses`, `/settlements` |
| Compound resource names | `kebab-case` | `/activity-logs`, `/group-members` |
| Route parameters | `camelCase` | `:groupId`, `:expenseId`, `:settlementId` |
| Query parameters | `camelCase` | `?sortBy=createdAt&sortOrder=desc&limit=20` |
| Nested resources | `/{parent}/{parentId}/{child}` | `/groups/:groupId/expenses` |

### 4.2 Action Endpoints

For non-CRUD operations that don't map cleanly to a resource:

| Pattern | Example | Description |
|---|---|---|
| `POST /{resource}/{id}/{verb}` | `POST /groups/:groupId/invite` | Send a group invitation |
| `POST /{resource}/{id}/{verb}` | `POST /groups/:groupId/archive` | Archive a group |
| `POST /auth/{verb}` | `POST /auth/register`, `POST /auth/login`, `POST /auth/refresh` | Auth actions |
| `POST /{resource}/{id}/{verb}` | `POST /users/me/export` | Trigger data export |

### 4.3 Filter & Search Conventions

```
GET /api/v1/groups/:groupId/expenses
  ?cursor=exp_abc123
  &limit=20
  &sortBy=createdAt
  &sortOrder=desc
  &category=food
  &payerId=usr_xyz
  &dateFrom=2026-01-01
  &dateTo=2026-03-31
  &search=dinner
```

- Filters are flat query parameters (`camelCase`).
- Date filters always in `YYYY-MM-DD` ISO format.
- Array filters use comma-separated values: `?category=food,transport`.

## 5. React Component Naming

### 5.1 Component Names

| Category | Convention | Examples |
|---|---|---|
| Page components | `PascalCase` with `Page` suffix | `DashboardPage`, `GroupDetailPage`, `LoginPage` |
| Feature components | `PascalCase`, descriptive | `ExpenseCard`, `SplitCalculator`, `BalanceList` |
| Layout components | `PascalCase`, layout-descriptive | `AppShell`, `Navbar`, `Sidebar`, `MobileNav` |
| Modal components | `PascalCase` with `Modal` suffix | `CreateGroupModal`, `AddExpenseModal`, `ConfirmSettlementModal` |
| Form components | `PascalCase` with `Form` suffix | `LoginForm`, `RegisterForm`, `SettleUpForm` |
| Shared/UI components | `PascalCase`, generic name | `Button`, `Input`, `Modal`, `Avatar`, `Badge`, `Spinner` |
| HOC wrappers | `PascalCase`, descriptive | `ProtectedRoute`, `ErrorBoundary` |
| Provider components | `PascalCase` with `Provider` suffix | `AuthProvider`, `SocketProvider`, `ThemeProvider` |

### 5.2 Component Props

| Convention | Pattern | Example |
|---|---|---|
| Props interface | `{ComponentName}Props` | `ExpenseCardProps`, `SplitCalculatorProps` |
| Event callbacks | `on{Event}` | `onSubmit`, `onClose`, `onExpenseCreated` |
| Boolean props | `is`, `has`, `show`, `enable` prefix | `isLoading`, `hasError`, `showDetails`, `enableEditing` |
| Render props | `render{Thing}` | `renderHeader`, `renderEmptyState` |
| Ref forwarding | `React.forwardRef` with `ref` prop | Standard React pattern |

### 5.3 Component File Structure

Every component file follows this internal order:

```
1. Imports (external → internal → styles)
2. Type definitions (Props interface)
3. Constants (local to the component)
4. Component function
5. Subcomponents (if any, private to the file)
6. Default export
```

## 6. CSS / Tailwind Naming

### 6.1 Custom CSS Classes (when needed beyond Tailwind)

| Convention | Pattern | Example |
|---|---|---|
| BEM for custom classes | `block__element--modifier` | `balance-card__amount--negative` |
| CSS custom properties | `--{namespace}-{property}` | `--ss-color-primary`, `--ss-spacing-md` |
| Animation keyframes | `camelCase` | `@keyframes fadeIn`, `@keyframes slideUp` |

### 6.2 Tailwind Usage

- Use Tailwind utility classes directly in JSX for all styling.
- Extract repeated patterns into Tailwind `@apply` directives in `globals.css` only for truly shared atoms (e.g., `.btn-primary`).
- Never mix inline `style={}` with Tailwind classes unless dynamic values require it.

## 7. Test Naming

### 7.1 Test File Placement

| Test Type | Location | Naming |
|---|---|---|
| Unit tests | `server/src/__tests__/unit/{layer}/` | `{module}.{layer}.test.ts` |
| Integration tests | `server/src/__tests__/integration/` | `{module}.integration.test.ts` |
| E2E tests | `server/src/__tests__/e2e/` | `{flow}.e2e.test.ts` |
| Component tests | Co-located with component | `{Component}.test.tsx` |

### 7.2 Test Description Conventions

```typescript
describe('ExpenseService', () => {
  describe('createExpense', () => {
    it('should create an expense with equal split among all participants', () => {});
    it('should reject when payer contributions do not equal total amount', () => {});
    it('should reject when fewer than 2 participants are specified', () => {});
    it('should emit a real-time event to the group room after creation', () => {});
  });

  describe('deleteExpense', () => {
    it('should allow deletion within the 48-hour window', () => {});
    it('should reject deletion when a referencing settlement exists', () => {});
    it('should reject deletion by a non-creator member', () => {});
  });
});
```

- `describe` blocks: noun (class/module name) → verb (method name).
- `it` blocks: start with `should` + expected behavior.
- Failure cases: `should reject when...` or `should throw when...`.

## 8. Git Conventions

### 8.1 Branch Naming

| Type | Pattern | Example |
|---|---|---|
| Feature | `feat/{ticket}-{short-description}` | `feat/SS-42-add-expense-split-calculator` |
| Bug fix | `fix/{ticket}-{short-description}` | `fix/SS-89-settlement-exceeds-balance` |
| Chore | `chore/{short-description}` | `chore/upgrade-prisma-5.11` |
| Hotfix | `hotfix/{short-description}` | `hotfix/refresh-token-rotation` |

### 8.2 Commit Message Format

```
{type}({scope}): {short description}

{optional body}

{optional footer: BREAKING CHANGE, fixes #issue}
```

**Types:** `feat`, `fix`, `refactor`, `test`, `docs`, `chore`, `perf`, `ci`  
**Scopes:** `auth`, `expenses`, `settlements`, `balances`, `groups`, `notifications`, `ui`, `db`, `config`

**Examples:**

```
feat(expenses): add percentage split mode with real-time preview
fix(settlements): prevent settlement amount exceeding outstanding balance (BR-05)
refactor(balances): extract debt simplification into shared utility
test(expenses): add integration tests for multi-payer scenarios
```
